# V5 Web Dashboard v2 - 重写计划

## 问题总结
- 现有代码过度复杂，多处重复逻辑
- 状态显示不一致（市场状态 vs 风险档位）
- 加载逻辑混乱，多个函数操作同一DOM
- 维护困难，改一处坏一处

## v2 设计原则
1. **单一数据源**：所有状态从 `auto_risk_guard.json` 和最新 `decision_audit.json` 读取
2. **模块化**：每个组件独立，不互相干扰
3. **极简UI**：只显示关键信息，去掉花哨效果
4. **稳定优先**：功能少但稳定，不再追求大而全

## 技术栈
- 纯HTML+JS，无框架
- 只使用 Fetch API 读取本地JSON
- 30秒自动刷新
- 错误时显示"数据加载失败"，不崩溃

## API端点
- `GET /api/account` - 账户信息
- `GET /api/positions` - 持仓列表
- `GET /api/trades` - 最近交易
- `GET /api/decision_audit` - 决策审计（含策略信号）
- `GET /api/auto_risk_guard` - 风险档位
