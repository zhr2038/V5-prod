# 2026-02-07 - V4 Trading Bot 调优记录

## 核心问题与解决方案

### 问题1：信号经常被验证器打掉导致0信号
**解决方案**：
1. 统一验证阈值口径：Validator 使用 `StrategyConfig.min_signal_strength` 校验 `signal.is_valid(min_strength)`
2. 均值回归信号强度不再被 *0.5 稀释，避免过不了0.3门槛
3. 趋势市允许均值回归辅助，避免regime过强约束导致信号长期为0
4. 增加 `signal_diag` 输出，无信号时显示last/momentum/rsi/boll位置，便于调参

### 问题2：下单失败（insufficient_usdt_free）
**解决方案**：
1. Portfolio allocator 的 notional 按 `trading.max_position_size/min_position_size` 做绝对cap
2. 主循环执行前按当前 `usdt_free-buffer` 截断BUY notional，防止余额变化导致失败
3. USDT释放目标改为 `min_pos + buffer`（默认buffer=2），避免刚补到5U又卡住

### 问题3：卖单"卖一点点"太蠢（碎片化交易）
**解决方案**：
1. 策略生成SELL时默认打 `indicators.sell_all=True`
2. ExecutionAdapter 看到SELL + sell_all，用 `exchange.fetch_balance()` 取base_free，按交易所精度卖出全部可用仓位

### 问题4：策略打架/翻来覆去（同一币同时buy/sell）
**解决方案**：
1. 增加position-aware bias：如果两种策略方向冲突且强度差<0.15
   - 有仓位优先SELL（先降风险/减仓）
   - 无仓位优先BUY（更像"开仓"逻辑）

### 问题5：手续费磨损/频繁交易
**解决方案**：
1. **BUY冷却**：`buy_cooldown_sec` 默认3600秒，同一币1小时内重复BUY跳过
2. **全局交易冷却**：`global_trade_cooldown_sec` 默认900秒，SELL永远允许，BUY在冷却期内跳过
3. **边际优势过滤**：
   - `rsi_edge = 2.0`：必须RSI < oversold-2才买；必须RSI > overbought+2才卖
   - `bollinger_edge = 0.001`：必须明显突破上下轨一定幅度才触发
