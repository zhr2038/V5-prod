## 下午发现问题：灰尘持仓阻塞交易

**现象**: 系统选币正常（DOT/UNI/NEAR/FIL/ADA），但连续多小时无实际交易

**根因分析**:
1. 账户有大量黑名单币灰尘持仓（SPACE/KITE/PEPE/PROMPT等），数量虽小但非零
2. 死区逻辑计算这些灰尘的偏离度，结果<3%，判定为"无需调整"
3. 实际可用资金被灰尘"占用"，无法建新仓

**09:00运行数据**:
- Selected: 5个币
- Orders rebalance: 4笔（生成）
- Deadband skip: 10笔（被拦截）
- 灰尘示例: SPACE(0.154), PEPE(0.233), PROMPT(0.0821)等

**尝试清理失败** (commit ed31b1b):
- 写脚本尝试市价卖出灰尘币
- 全部10个币因数量低于OKX最小下单量而无法卖出
- OKX App也无"小额资产兑换"功能

**最终解决方案** (commit 5554306):
- 改代码：pipeline中添加灰尘持仓过滤
- 阈值: 数量<0.01 或 价值<$1 视为无持仓
- 灰尘会被记录到audit日志但不会阻塞死区判断

**配置变更**:
```python
DUST_QTY_THRESHOLD = 0.01     # 数量阈值
DUST_VALUE_THRESHOLD = 1.0    # 价值阈值(USD)
```
